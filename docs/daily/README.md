# Daily task briefs

This folder contains one markdown per day (`dayNN.md`) covering the full 30-day plan in `docs/plan/PLAN.md`.

Each brief follows the same structure:

1. **Why this day matters** — one paragraph linking the day to the job-search outcome
2. **Reading (1–2 links max)** — curated, no information overload
3. **Build tasks** — exact files to create / modify, with code skeletons or interface sketches
4. **Acceptance criteria** — how you know the day is done
5. **Commit message** — what to write
6. **If you finish early** — stretch goals
7. **If you fall behind** — what to cut

Day numbering aligns with the plan:

| Week | Days | Theme |
|---|---|---|
| W1 | 01–07 | Agent basics + Python async + reading source code |
| W2 | 08–14 | Sandbox (C++) + MCP tool server |
| W3 | 15–21 | Agent loop + multi-provider + trajectories |
| W4 | 22–30 | SWE-bench + résumé + applications |

Day 01 is already done (`../notes/day01.md` for the actual log). Day 02 onward is what to follow next.

**Rule:** read this brief in the morning, attack the tasks, log to `docs/notes/dayNN.md` at night.
